自定义组件样式

组件样式隔离 注意点

app.wxss全局样式对组件无效
只有clas选择器有样式隔离！ id选择器，属性选择，标签选择器不受样式隔离影响
在组件和引用组件页面韩总 建议用class选择器，不用 id 属性 标签选择器

修改 组件样式隔离 选项

外部控制内部样式： 通过stylesolation
isolated 启用样式隔离 （默认）
apply-shared 页面影响 自定义组件
shared 双向影响

  • ① 在组件 .js 文件新增配置
  • ② 在组件的 .json文件新增配置

data 数据

在小程序组件中，用于组件末班渲染的私有数据，需要定义到data节点中

methods 方法

  • 在小承租组件中，事件处理函数和自定义方法需要定义到 methods 节点中

properties 属性

  • 在小程序组件中，properties是组件的对外属性，用来接受外界传递到组件的数据

data和properties区别
data 倾向存组件的私有数据 可读可写
properties 倾向存组件的 外界来的数据 可读可写

自定义组件 - 数据监听器

渲染UI结构

监听对象中指定属性变化

监听对象中所有属性的变化

纯数据字段

  • 概念：纯数据字段指的是哪些不用于界面渲染的data字段
  • 场景： 某些data的字段 既不会展示在界面上，也不会传递给其他组件，仅在组件内部使用，带这猴子那个特性的data字段适合被设置为纯数据字段
  • 有助于提升页面性能
  • 使用规则： 在Component构造器的options节点中，指定pureDataPattern为正则表达式，字段名符合这个规则表达式的字段将成为纯数据字段

使用纯数据字段改在数据监听器

组件的生命周期

组件的全部生命周期函数

生命周期函数 参数 说明
created 无 在组件实例刚刚被创建时候执行
attached 无 在组件实例进入页面节点数时执行
ready 无 在组件在视图层布局完成后执行
moved 无 在组件被移动盗节点另外一个位置时执行
detached 无 在组件实例被从页面节点数移除时候执行
eddor Object Error 每当组件方法抛出错误时候执行

①created

  • 不能调用setData
  • 在这个函数周期内，只应该用于给组件得this 挂载一些自定义的属性字段

    ②attached 完全初始化玩不，进入页面节点树后 出发attached函数

  • this.data已被初始化完毕
  • 大部分初始化工作在此函数，eg:发请求获取初始化数据

    ③detached 在组件页面离开页面节点树后，detached函数触发

  • 退出一个面，会触发页面内每一个自定义组件detached函数触发
  • 清理工作，eg:销毁时间监听

lifttimes 节点

  • 在小程序组件中，生命周期函数可以直接定义在Component构造器的第一级参数中，可以在lifttimes字段内进行声明（这是推的方式，其优先级最高）

组件所在页面的生命周期

  • 自定义组件的行为需要依赖页面状态的变化，则就徐亚用到组件所在页面的生命周期
生命周期函数 参数 说明
show 无 组件所在页面 被展示时候执行
hide 无 组件所在页面 被影藏时候执行
resize Object Size 组件所在页面 尺寸发生变化时候执行

pageLifrtimes 节点

  • 组件所在页面的生命周期函数，需要定在pageLifrtimes节点中

自定义组件 - 插槽

  • 在自定义组件wxml结构中，提供一个节点（插槽），用于承载组件使用者提供的wxml结构

单个插槽

  • 在小程序中，默认每个自定义组件只允许使用一个进行占位，这种个数上的限制叫单个插槽

组件的封装者-

组件得使用者

启用多个插槽

配置 - 在小程序的自定义组件还在那个，需要使用多插槽时，在组件的.js文件中，在options节点 设置mutipleSlots

构建 - 组件的.wxml中使用多个标签，以不同的name来区分不同插槽

使用 -带有多个插槽的自定义组件时，需要用slot属性来将节点插入到不同的中

父子组件通信

父子组件通信 -方式

① 属性绑定

  • 父组件->子组件的指定属性设置数据，仅能设置json兼容的数据

    ② 事件绑定

  • 子组件->父组件传递数据，传递任意数据

    ③ 获取组件实例

  • 父组件中，通过this.selectComponent()获取子组件实例对象
  • 通过实例对象，直接访问子组件任意数据和方法

① 属性绑定 （父传子）

  • 属性绑定用于实现 （父组件->子组件），只能传递普通类型的数据，不能传递方法

父组件

子组件

  • 子组件在properties节点中声明对应的属性并使用

② 事件绑定（子传父）

传递任意类型数据

  • 在父组件的js中，定义一个函数，这个函数通过自定义事件形式，传递给子组件
  • 在父组件的.wxml中，通过自定义事件的形式，将步骤1的函数引用，传递给子组件
  • 在子组件的js种，通过调用this.triggerEvent(‘自定义事件名称’, {参数对象})，将数据发送到父组件
  • 在父组件的js中，通过e.detail获取到子组件传递过来的数据

③ 获取组件实例

  • 父组件调用this.selectComponent(id或class选择器)，获取子组件的实例对象，从而直接访问子组件任意数据和方法，
  • 调用时候需要传入一个选择器 eg： this.selectComponent(“.my-component”)

自定义组件 - behaviors

behavior 是小程序中，用于实现组件代码贡献的特性，（vue.js的mixins）

behavior - 工作方式

每个 behaviors 可以包含一组属性 数据 生命周期函数 和方法，组件应用它时，他的属性 数据 方法 会被合并到组件中
每个组件可以引用多个 behavior， behaviors可以用引用其它 behavior

behavior - 创建

调用Behavior(Object object)方法 创建一个共享 behavior 实例对象

behavior - 导入 + 使用

在组件中，使用require()方法导入需要的behavior，挂载后即可访问behavior的数据或方法

behavior - 可用节点

可用节点 type 是否必填 描述 其他
properties Object Map 同组件的属性
data Object 同组件的数据
methods Object 同自定义组件的方法
behaviors String Array 引入其他的behavior
created Function leftCycle函数
attached Function leftCycle函数
ready Function leftCycle函数
moved Function leftCycle函数
detached Function leftCycle函数

behavior - 同名字段覆盖和组合规则

组件和他引用的behavior可以包含同名的字段
处理规则：

  • ①data同名的数据字段
  • ②同名的属性properties 或方法 methods
  • ③同名的生命周期函数
